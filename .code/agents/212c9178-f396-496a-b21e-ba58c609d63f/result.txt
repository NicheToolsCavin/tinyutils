**Key Assumptions / Questions**
- Inputs remain blob-backed via existing `tinyutils/api/_lib/blob.py`; no direct file uploads.
- Downstream `/tools/convert` UI (see `tools/convert/index.html`) expects the current envelope shape; we can extend metadata but must not break existing keys.
- Multi-target requests share identical source inputs; assume we can reuse extracted media between targets rather than re-running Pandoc per target? (TBD).

**Proposed Architecture**
- Introduce `tinyutils/convert/` package: 
  - `batch.py` orchestrates batches (multiple inputs × targets) and normalizes options/metadata.
  - `runner.py` wraps Pandoc + Lua filters using `tinyutils/api/_lib/pandoc_runner.py`, emitting structured logs/events.
  - `manifest.py` (or reuse `_lib/manifests.py`) for deterministic metadata (headings/snippets/images) per converted asset.
  - `outputs.py` handles blob upload coordination via `_lib/blob.py`, plus optional ZIP bundling for media.
- Expose a façade `tinyutils/convert/service.py` consumed by `tinyutils/api/convert/index.py` so the API layer mainly validates request/envelope plumbing.
- Central log collector to capture per-file/per-target steps, feeding previews and `logs` in response.
- Shared `convert/models.py` for typed request/response schemas so validators/testing align.

**Step-by-Step Plan (with rationale)**

3. **Implement Batch Orchestrator** – Build `tinyutils/convert/batch.py` that iterates inputs × targets, invokes runner, aggregates logs/errors, and handles early abort vs per-file failure policy. Ensure media extraction is keyed per input and reused for all targets.
4. **Abstract Conversion Runner** – Move Pandoc/Lua pipeline into `tinyutils/convert/runner.py`, wrapping `_lib/pandoc_runner.py` and ensuring consistent tool version reporting. Include hooks for metadata extraction (`_lib/text_clean.py`, `_lib/manifests.py`).
5. **Blob & Artifact Strategy** – Centralize blob upload + artifact writing in `tinyutils/convert/outputs.py`, covering: per-output blob upload, optional ZIP for extracted media, and dated artifact mirrors under `tinyutils/artifacts/convert/<date>/`. Ensure uploads return size + URL for envelope.
6. **API Layer Update** – Refactor `tinyutils/api/convert/index.py` to delegate to the new service, preserving envelope structure (jobId/toolVersions/outputs/preview/logs/errors) while now supporting multi-file/multi-target.
7. **Metadata Determinism** – Extend `_lib/manifests.py` or new helper so each converted document yields headings/snippets/images in a predictable order; include this metadata in outputs to support future UI enhancements.
8. **Logging & Docs** – Update `docs/AGENT_RUN_LOG.md` and `docs/AGENT_TASK_CHECKLIST.md` to reflect new module boundaries, configuration knobs, and operational steps. Ensure log messages in service expose file/target context for easier debugging.
9. **Test Coverage & Fixtures** – Add fixtures under `tinyutils/tests/fixtures/convert/` (single doc, multi-file, binary + media). Write regression tests under `tinyutils/tests/convert/` covering single input, batch, multi-target, and error handling; verify blob upload stubs and metadata outputs.
