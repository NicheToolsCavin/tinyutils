Here’s a precise, single‑request Blob validation recipe for preview that captures evidence and yields a clear pass/fail.

**Purpose**
- Prove preview returns `https://*.blob.vercel-storage.com/...` URLs (not `data:` URIs).
- Probe returned URL with HEAD for HTTP 200.
- Persist evidence under `tinyutils/artifacts/convert/<date>/blob-validate-*` and summarize in `docs/AGENT_RUN_LOG.md`.

**Required Envs**
- `BLOB_READ_WRITE_TOKEN` (Vercel Blob RW token).
- `PREVIEW_BASE_URL` (e.g., `https://<your-preview>.vercel.app`).
- `CONVERT_PATH` (path to conversion endpoint, e.g., `/api/convert`).
- Optional tuning: `CURL_OPTS` (e.g., `-sS`), `REQUEST_TIMEOUT` (seconds, default 30).

**One-Shot Script**
- Run this from repo root `/Users/cav/dev/TinyUtils` once preview 500s are resolved.
- It sends one conversion request using a public tiny PNG, asserts the returned URL shape, HEADs the URL, and saves artifacts.

```
#!/usr/bin/env bash
set -euo pipefail

: "${BLOB_READ_WRITE_TOKEN:?missing}"  # presence check
: "${PREVIEW_BASE_URL:?missing}"
CONVERT_PATH="${CONVERT_PATH:-/api/convert}"
CURL_OPTS="${CURL_OPTS:--sS}"
REQUEST_TIMEOUT="${REQUEST_TIMEOUT:-30}"

DATE_UTC="$(date -u +%F)"
RUN_ID="blob-validate-$(date -u +%Y%m%dT%H%M%SZ)"
OUTDIR="tinyutils/artifacts/convert/${DATE_UTC}"
mkdir -p "${OUTDIR}"

REQ_FILE="${OUTDIR}/${RUN_ID}-request.json"
RESP_FILE="${OUTDIR}/${RUN_ID}-response.json"
URL_FILE="${OUTDIR}/${RUN_ID}-blob-url.txt"
HEAD_FILE="${OUTDIR}/${RUN_ID}-head.txt"
RESULT_FILE="${OUTDIR}/${RUN_ID}-result.txt"
SUMMARY_FILE="${OUTDIR}/${RUN_ID}-summary.md"

# 1) Build request (adjust fields to your API’s contract)
# Assumption: your endpoint uploads result to Blob and returns a URL in one of: .url | .output_url | .blob.url
cat > "${REQ_FILE}" <<'JSON'
{
  "input_url": "https://httpbin.org/image/png",
  "format": "png"
}
JSON

# 2) One conversion request (expect Blob URL in response JSON)
CONVERT_URL="${PREVIEW_BASE_URL}${CONVERT_PATH}"
curl ${CURL_OPTS} --fail -X POST \
  -H "Content-Type: application/json" \
  --max-time "${REQUEST_TIMEOUT}" \
  --data @"${REQ_FILE}" \
  "${CONVERT_URL}" | tee "${RESP_FILE}" >/dev/null

# 3) Extract output URL; accept a few common shapes
URL="$(jq -r '.url // .output_url // .blob.url // empty' "${RESP_FILE}")"

# 4) Validate URL shape (must be https://*.blob.vercel-storage.com/..., not data:)
if [[ -z "${URL}" ]]; then
  echo "FAIL: no URL in response" | tee "${RESULT_FILE}"
  exit 1
fi
echo "${URL}" > "${URL_FILE}"

if [[ "${URL}" =~ ^data: ]]; then
  echo "FAIL: got data: URI (expected Blob HTTPS URL)" | tee -a "${RESULT_FILE}"
  exit 1
fi

if ! [[ "${URL}" =~ ^https://[^/]+\.blob\.vercel-storage\.com/.+ ]]; then
  echo "FAIL: URL not in *.blob.vercel-storage.com: ${URL}" | tee -a "${RESULT_FILE}"
  exit 1
fi

# 5) HEAD probe for 200 (follow redirects)
STATUS_LINE="$(curl ${CURL_OPTS} -I -L --max-time "${REQUEST_TIMEOUT}" "${URL}" | tee "${HEAD_FILE}" | head -n1)"
HTTP_CODE="$(sed -n 's/^HTTP\/[0-9.]* \([0-9][0-9][0-9]\).*$/\1/p' <<<"${STATUS_LINE}")"

if [[ "${HTTP_CODE}" != "200" ]]; then
  echo "FAIL: HEAD status ${HTTP_CODE} (expected 200)" | tee -a "${RESULT_FILE}"
  exit 1
fi

# 6) PASS summary
{
  echo "PASS"
  echo "url=${URL}"
  echo "http=${HTTP_CODE}"
} | tee "${RESULT_FILE}"

# 7) Human summary (for quick inspection)
cat > "${SUMMARY_FILE}" <<EOF
Blob Validation PASS
- Preview: ${PREVIEW_BASE_URL}
- Endpoint: ${CONVERT_PATH}
- URL: ${URL}
- HEAD: ${HTTP_CODE}
- Run ID: ${RUN_ID}
EOF

# 8) One-line runbook entry for docs/AGENT_RUN_LOG.md (append)
echo "$(date -u +%Y-%m-%dT%H:%M:%SZ) blob-validate preview=${PREVIEW_BASE_URL} status=PASS http=${HTTP_CODE} url=${URL} out=${OUTDIR}/${RUN_ID}-*" >> docs/AGENT_RUN_LOG.md
```

**Artifacts Saved**
- `tinyutils/artifacts/convert/<date>/<run-id>-request.json` (exact request payload).
- `tinyutils/artifacts/convert/<date>/<run-id>-response.json` (raw response).
- `tinyutils/artifacts/convert/<date>/<run-id>-blob-url.txt` (extracted URL).
- `tinyutils/artifacts/convert/<date>/<run-id>-head.txt` (HEAD response headers; first line includes status).
- `tinyutils/artifacts/convert/<date>/<run-id>-result.txt` (`PASS`/`FAIL`, plus details).
- `tinyutils/artifacts/convert/<date>/<run-id>-summary.md` (human-readable recap).

**Pass/Fail Criteria**
- PASS when:
  - Response contains a URL and it matches `^https://[^/]+\.blob\.vercel-storage\.com/.+`.
  - URL does not start with `data:`.
  - HEAD returns HTTP 200.
- FAIL on any deviation (missing URL, wrong scheme/host, HEAD ≠ 200).

**Runbook Entry**
- Format: `YYYY-MM-DDTHH:MM:SSZ blob-validate preview=<url> status=<PASS|FAIL> http=<code|NA> url=<blob-or-reason> out=tinyutils/artifacts/convert/<date>/<run-id>-*`
- Example: `2025-11-10T17:34:22Z blob-validate preview=https://tinyutils-preview.vercel.app status=PASS http=200 url=https://abc.blob.vercel-storage.com/xyz out=tinyutils/artifacts/convert/2025-11-10/blob-validate-20251110T173422Z-*`

If your convert endpoint uses different fields, adjust the `request.json` shape and the `jq` extraction (`.url // .output_url // .blob.url`) to match. Want me to tailor the request for your actual endpoint signature?
