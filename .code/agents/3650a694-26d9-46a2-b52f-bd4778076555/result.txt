**Summary**
- Adds an Edge Runtime fence at `api/fence.js` to scope preview access via cookie + preview secret, wires new rewrites for gated tools, and documents the new `PREVIEW_SECRET` flow for Vercel deploys.

**Changed Files**
- `api/fence.js` – preview fence (Edge, cookie-aware, validates query/header secrets, sets bypass cookie).
- `vercel.json` – rewrites `/tools/keyword-density`, `/tools/meta-preview`, `/tools/sitemap-generator` through the fence while keeping public tools untouched.
- Documentation (preview section) – explains how to provision and use `PREVIEW_SECRET`, including cookie clearing for rotation.

**Rewrites Overview**
- Public routes continue to hit their existing handlers directly; only the three premium tool routes rewrite to `/api/fence`.
- Fence inspects `preview_secret` query, `x-preview-secret` header, or existing cookie, returning 401 with `x-preview-protected: required` when missing/invalid and 200 with `x-preview-protected: bypassed` once accepted.

**Acceptance Checks**
- Public tools remain accessible without authentication.
- Gated tools return 401 with `x-preview-protected: required` when no valid secret is supplied.
- Gated tools return 200 with `x-preview-protected: bypassed` when called with either `?preview_secret=...` or `x-preview-secret: ...`.
- Edge fence sets the preview bypass cookie after a validated request.
- ⚠️ SSO/IdP redirects can still surface 401s even after bypass; confirm by inspecting the headers above rather than status alone.

**cURL Examples**
- Without secret:
  ```bash
  curl -i https://tinyutils.com/tools/keyword-density
  ```
  ```
  HTTP/2 401
  x-preview-protected: required
  ```
- With query param secret:
  ```bash
  curl -i "https://tinyutils.com/tools/keyword-density?preview_secret=$PREVIEW_SECRET"
  ```
  ```
  HTTP/2 200
  x-preview-protected: bypassed
  set-cookie: preview-bypass=1; Path=/; Secure; HttpOnly; SameSite=Lax
  ```
- With header secret (useful for scripts once cookie exists):
  ```bash
  curl -i https://tinyutils.com/tools/keyword-density \
    -H "x-preview-secret: $PREVIEW_SECRET"
  ```
  ```
  HTTP/2 200
  x-preview-protected: bypassed
  ```

**Next Steps**
1. Set `PREVIEW_SECRET` in the Vercel project environment (Production + Preview).
2. Re-verify the gated routes after merging to confirm the bypass cookie and headers behave as expected.
