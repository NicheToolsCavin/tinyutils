## TinyUtils Reconciliation Plan — 2025-11-04 (Execution Ready)

This plan sequences dependencies, outlines environment prerequisites, enumerates validation/evidence expectations, and surfaces risk mitigations. All logging and documentation will use CET timestamps and leverage `scripts/log_run_entry.py` and `scripts/add_task_checklist_entry.py`. Evidence will be stored under `artifacts/<task>/<YYYYMMDD>/` and supporting diffs in `diffs_recs/`.

### Phase 0: Setup & Baseline

**Objective:** Establish a stable working environment and ensure foundational elements are in place.

*   **Task 0.1: Git/Log Backfill**
    *   **Description:** Ensure local git history is up-to-date and review recent logs for context.
    *   **Prerequisites:** Git installed.
    *   **Validation/Evidence:** `git status`, `git log -n 10`.
    *   **Logging/Documentation:** Record current git state and any relevant log entries in `docs/AGENT_RUN_LOG.md`.

*   **Task 0.2: Orchestrator Agent Roster Check**
    *   **Description:** Verify `AGENTS.md` and `config.toml` reflect the current orchestrator roster (code-gpt-5, code-gpt-5-codex, 3 hacked ChatGPT handles, Gemini fallback).
    *   **Prerequisites:** Access to `AGENTS.md` and `config.toml`.
    *   **Validation/Evidence:** Manual review of file contents.
    *   **Logging/Documentation:** Confirm roster accuracy in `docs/AGENT_TASK_CHECKLIST.md`.

*   **Task 0.3: Environment Prerequisites Check**
    *   **Description:** Confirm Vercel deployment access, `pnpm` installation, `curl` availability, and functionality of custom scripts (`scripts/`).
    *   **Prerequisites:** Vercel CLI configured, `pnpm` installed, `curl` available.
    *   **Validation/Evidence:** Run `vercel --version`, `pnpm --version`, `curl --version`, and execute a simple test script (e.g., `node scripts/preview_smoke.mjs --help`).
    *   **Logging/Documentation:** Record versions and successful script execution in `docs/AGENT_RUN_LOG.md`.

### Phase 1: PR1 - Security Headers & `/public` Caching

**Objective:** Implement and verify security headers, specifically Content Security Policy (CSP), and ensure correct caching for `/public` assets.

*   **Task 1.1: Implement CSP**
    *   **Description:** Add or update Content Security Policy (CSP) headers in `vercel.json` to enhance security.
    *   **Prerequisites:** Understanding of existing `vercel.json` structure.
    *   **Validation/Evidence:** Deploy to a preview environment. Use `curl -I <preview-url>` to capture response headers. Analyze headers for correct CSP implementation. Store header captures in `artifacts/pr1-security-caching/<YYYYMMDD>/`.
    *   **Risk Mitigation:** **CSP Drift:** Carefully review CSP directives to avoid breaking existing functionality. Start with a report-only mode if possible, or a very permissive policy, and gradually tighten.
    *   **Logging/Documentation:** Record `vercel.json` changes in `diffs_recs/pr1-security-caching.diff`. Add entry to `docs/AGENT_TASK_CHECKLIST.md` for CSP implementation and verification.

*   **Task 1.2: Verify `/public` Caching**
    *   **Description:** Confirm that assets served from the `/public` directory have appropriate caching headers.
    *   **Prerequisites:** Deployed preview environment.
    *   **Validation/Evidence:** Use `curl -I <preview-url>/public/styles.css` (or another static asset) to capture response headers. Verify `Cache-Control` and `Expires` headers. Store header captures in `artifacts/pr1-security-caching/<YYYYMMDD>/`.
    *   **Logging/Documentation:** Add entry to `docs/AGENT_TASK_CHECKLIST.md` for caching verification.

### Phase 2: PR2 - Noindex Meta & Debug Hook

**Objective:** Implement `noindex` meta tags for specific pages and add a debug hook for controlled debugging.

*   **Task 2.1: Implement Noindex Meta Tags**
    *   **Description:** Add `<meta name="robots" content="noindex">` to pages that should not be indexed by search engines (e.g., internal tools, temporary pages).
    *   **Prerequisites:** Identify target pages (e.g., `feedback.html`, `support.html` if not public-facing).
    *   **Validation/Evidence:** Deploy to a preview environment. Use `curl <preview-url>/feedback.html` and inspect the HTML content for the `noindex` meta tag. Store HTML snippets in `artifacts/pr2-ux-noindex-debug/<YYYYMMDD>/`.
    *   **Logging/Documentation:** Record file changes in `diffs_recs/pr2-ux-noindex-debug.diff`. Add entry to `docs/AGENT_TASK_CHECKLIST.md`.

*   **Task 2.2: Implement Debug Hook**
    *   **Description:** Introduce a controlled debug hook (e.g., a query parameter or environment variable) to enable/disable debug logging or features.
    *   **Prerequisites:** Identify appropriate location for the hook (e.g., `api/check.js` or a shared utility).
    *   **Validation/Evidence:** Test the hook in a preview environment by activating and deactivating it, observing expected debug output or behavior.
    *   **Logging/Documentation:** Record code changes in `diffs_recs/pr2-ux-noindex-debug.diff`. Add entry to `docs/AGENT_TASK_CHECKLIST.md`.
