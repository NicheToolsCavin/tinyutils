```markdown
## Pandoc Path Resolution and Fallback Behavior

### Path Resolution Priority

The system resolves the pandoc executable path in the following order:

1. **`PYPANDOC_PANDOC` Environment Variable** (optional override)
   - If set, overrides all other paths
   - Expected to point to a valid, executable pandoc binary
   - Warning logged if path is invalid or not executable

2. **Vendored Pandoc Executable**
   - Located at `tinyutils/api/_lib/vendor/pandoc`
   - Bundled with the package for guaranteed availability
   - Selected automatically if present and executable

3. **System Pandoc in PATH**
   - Standard pandoc installation accessible via system PATH
   - Used as final fallback when other options are unavailable

### Fallback Semantics

- When a pandoc executable is found, it's used for conversion operations
- Health checks log the active pandoc executable at INFO level: `"Pandoc health check passed: {path}"`
- If no pandoc executable is found after checking all locations, conversion operations raise `RuntimeError`
- The system validates executable accessibility before each conversion

### Startup Health Logging

- Upon first conversion attempt, the system performs a health check
- Logs which pandoc executable is being used for monitoring purposes
- Example: `INFO - Pandoc health check passed: /path/to/pandoc`
```
