+### Smoke-only preview checks
+
+If you already have a deployed preview and only need to re-run the smoke without touching Vercel, export the preview URL and set `SMOKE_ONLY=1`. Any fence header/cookie you provide will be echoed in the logs and persisted inside `tinyutils/artifacts/convert/<date>/smoke-*`.
+
+```sh
+SMOKE_ONLY=1 \
+PREVIEW_URL="https://tinyutils-preview-example.vercel.app" \
+PREVIEW_FENCE_HEADER="x-vercel-protection-bypass" \
+PREVIEW_FENCE_VALUE="$BYPASS_TOKEN" \
+PREVIEW_FENCE_COOKIE="__secure_next_preview_data" \
+PREVIEW_FENCE_COOKIE_VALUE="$BYPASS_COOKIE" \
+bash tinyutils/scripts/deploy_and_smoke_convert.sh
+```
+
+When `SMOKE_ONLY` is unset (the default), the script deploys a fresh preview first and then calls `smoke_convert_preview.mjs`, which now records the request/response pair under the dated artifact directory for later inspection.
