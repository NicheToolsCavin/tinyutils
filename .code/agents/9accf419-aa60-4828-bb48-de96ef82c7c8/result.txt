```yaml
        run: |
          mkdir -p .vercel
          pnpm exec vercel env pull .vercel/.env.preview.local --environment=preview --yes
        env:
          VERCEL_TOKEN: ${{ secrets.VERCEL_TOKEN }}
          VERCEL_ORG_ID: ${{ secrets.VERCEL_ORG_ID }}
          VERCEL_PROJECT_ID: ${{ secrets.VERCEL_PROJECT_ID }}

      - name: Convert preview smoke
        env:
          CONVERT_SMOKE_ARTIFACTS: artifacts/convert/github-${{ github.run_id }}
        run: |
          set -a
          source .vercel/.env.preview.local
          set +a
          pnpm smoke:convert:preview

      - name: Upload convert artifacts
        if: always()
        uses: actions/upload-artifact@v4
        with:
          name: convert-preview-smoke
          path: artifacts/convert
          if-no-files-found: ignore
```

**Secrets & Permissions**
- GitHub secrets must include `VERCEL_TOKEN`, `VERCEL_ORG_ID`, and `VERCEL_PROJECT_ID` for the CLI pull; the Vercel preview env should already define `PREVIEW_URL`, any required `PREVIEW_FENCE_HEADER`/bypass tokens, and `PREVIEW_SECRET` so the smoke can authenticate.
- Workflow only needs the default `contents: read` permission set above.
