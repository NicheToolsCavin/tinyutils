Here’s a minimal, low‑risk upgrade plan to Python 3.12 with FastAPI ≥0.110 and Pydantic v2, focused on the convert API models in `tinyutils/api/convert/app.py` and tests in `tests/convert/`.

**Version Pins**
- Runtime: Python 3.12.x
- fastapi==0.110.0
- pydantic==2.6.4
- starlette==0.37.2
- uvicorn==0.25.0
- pydantic-settings==2.2.1 (only if `BaseSettings` is used)
- httpx==0.27.0 (if your tests use it)

**Key Code Changes (Pydantic v2)**
- Imports
  - Replace `from pydantic import BaseModel, Field, validator, root_validator` with:
    - `from pydantic import BaseModel, Field, ConfigDict, field_validator, model_validator`
- Config → model_config
  - Replace inner `class Config:` with `model_config = ConfigDict(...)`
  - Common mappings:
    - `allow_population_by_field_name=True` → `populate_by_name=True`
    - `arbitrary_types_allowed=True` → same key in `ConfigDict`
    - If using env settings:
      - `from pydantic_settings import BaseSettings, SettingsConfigDict`
      - `model_config = SettingsConfigDict(env_file=".env", env_prefix="...")`
- Validators
  - `@validator("field", pre=True)` → `@field_validator("field", mode="before")`
  - `@validator("field")` → `@field_validator("field")`
  - `@root_validator(pre=True)` → `@model_validator(mode="before")`
  - `@root_validator` → `@model_validator(mode="after")`
- Serialization/Parsing
  - `.dict(...)` → `.model_dump(...)`
  - `.json(...)` → `.model_dump_json(...)`
  - `Model.parse_obj(data)` → `Model.model_validate(data)`
  - `Model.parse_raw(json_str)` → `Model.model_validate_json(json_str)`
  - `Model.schema()` → `Model.model_json_schema()`
- Aliases and Field constraints
  - `Field(..., alias="foo")` still valid; ensure `populate_by_name=True` if mixing alias/name inputs
  - `Field(..., regex="...")` → `Field(..., pattern="...")`
- Custom encoders
  - `json_encoders={Type: encoder}` → use `@field_serializer("field")` or `@model_serializer` as needed
- Generics and Private attrs
  - `GenericModel` import path is `from pydantic import BaseModel` + typing generics (or `pydantic.generics` if used)
  - `PrivateAttr` unchanged

Apply the above in `tinyutils/api/convert/app.py` specifically where models/validators/config are defined and where `.dict()/.json()/parse_*()` are used in request/response shaping.

- `jsonable_encoder` remains available; often you can remove it if returning Pydantic v2 models directly.

  - `.model_dump(by_alias=True)` used anywhere responses are built manually

  - Regex/pattern validation still triggers appropriately
- Schema

- Parsing helpers
  - Any internal uses of `parse_obj/parse_raw/dict/json/schema` updated and covered

  - Env overrides load via `pydantic-settings` with `SettingsConfigDict`
