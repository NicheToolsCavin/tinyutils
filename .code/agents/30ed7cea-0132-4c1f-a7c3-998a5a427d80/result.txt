**Recommended Pins (Py 3.11, Linux x86_64)**
- fastapi==0.115.5
- pydantic==2.9.2
- pydantic-core==2.23.4
Why: These versions are stable together, FastAPI fully supports Pydantic v2, and pydantic-core publishes cp311 manylinux wheels, avoiding source builds on Vercel’s Linux x64.

**Pandoc Choice**
- If you vendor your own pandoc binary (checked into the repo or provided by the runtime image/Layer), switch to `pypandoc` (not `pypandoc-binary`).
- Reason: `pypandoc-binary` bundles a platform-specific pandoc and can bloat wheels or mismatch glibc; with a vendored pandoc you get smaller installs and fewer platform surprises.
- Ensure pandoc is discoverable via `PATH` or set `PYPANDOC_PANDOC=/path/to/pandoc`.

**Minimal requirements.txt Change**
- Replace `pypandoc-binary` with `pypandoc`.
- Pin FastAPI/Pydantic stack to versions with manylinux wheels.

Example:
```
fastapi==0.115.5
pydantic==2.9.2
pydantic-core==2.23.4
pypandoc
```

**Rationale**
- manylinux wheels: pydantic-core provides prebuilt cp311 manylinux_x86_64 wheels for these versions, preventing build failures on Vercel.
- Compatibility: FastAPI 0.115.x targets Pydantic v2; pydantic 2.9.2 is within the supported range and matches pydantic-core 2.23.x.
- Pandoc: Using `pypandoc` with a vendored binary avoids unnecessary bundled binaries and reduces the chance of glibc issues.
