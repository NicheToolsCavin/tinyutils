Why iPhones Use HEIC (And Why It's a Problem)

Since iOS 11 (2017), iPhones save photos in HEIC (High Efficiency Image Container) format by default. Here's why Apple made this choice:

  • 50% smaller files: A 12MP photo that's 3MB as JPEG is only ~1.5MB as HEIC
  • Better quality: HEIC preserves more detail at the same file size
  • Storage savings: Your 256GB iPhone effectively holds twice as many photos

The problem? Browsers can't display HEIC.

HEIC uses the HEVC (H.265) codec, which requires patent licenses. Browser vendors (Google, Mozilla, Apple for web) won't pay those fees, so HEIC remains unsupported on the web.

The irony: Apple's Safari browser can't display photos taken by Apple's iPhone camera in its native format on websites.

Bottom line: WebP gives you near-HEIC compression with universal browser support. It's the ideal target format for iPhone photos destined for the web.

What Happens During Conversion

Here's the technical flow when you convert HEIC to WebP with our tool:

1. HEIC Decoding

The heic2any library (loaded on-demand, only when needed) decodes your HEIC file into raw pixel data. This happens entirely in your browser — the 332KB library is never loaded unless you actually drop a HEIC file.

2. Optional Resize

If you've enabled resizing, the image is scaled using a multi-step downscaling algorithm for maximum quality. A 4000px photo scaled to 1200px goes through multiple intermediate sizes to preserve detail.

3. WebP Encoding

The browser's native WebP encoder (via OffscreenCanvas) compresses the image. This uses your CPU, not a remote server, so processing speed depends on your device.

4. Download

The WebP file is created as a Blob in memory. When you click download, you're saving a file that was created locally — nothing was ever uploaded.

Why "No Upload" Matters for Photo Conversion

When you upload photos to conversion services, you're trusting them with:

  • Your images: Personal photos, potentially with faces, locations, private moments
  • EXIF metadata: GPS coordinates, camera settings, timestamps
  • Implicit consent: Many free services use uploaded images for AI training

With client-side conversion, none of this is a concern. Your photos stay on your device. We literally cannot see them.

Verification: Open your browser's Network tab (F12 → Network) while converting. You'll see zero upload requests. The HEIC decoding library downloads once, then everything happens locally.

Troubleshooting HEIC Conversion

"HEIC file not recognized"

Some HEIC files have unusual extensions like .HEIF or no extension at all. Rename to .heic and try again.

"Conversion is slow"

HEIC decoding is CPU-intensive. On older devices, a 12MP photo might take 2-3 seconds. This is normal — the trade-off for privacy is using your local CPU instead of a server farm.

"Output looks different from original"

HEIC can store HDR (High Dynamic Range) data that WebP doesn't fully support. The conversion maps HDR to SDR, which may slightly change the look of high-contrast scenes.

"Can I convert back to HEIC?"

No — we only support HEIC as input, not output. Browsers can't encode HEIC due to patent restrictions. If you need HEIC output, you'll need desktop software like Apple's native apps or Adobe tools.

HEIC vs WebP vs JPEG: When to Use Each

Keep HEIC when...

Storing on Apple devices, archiving originals, using in iOS/macOS apps

Convert to WebP when...

Publishing to web, need transparency, want smallest files with modern browser support

Convert to JPEG when...

Maximum compatibility required (email, legacy systems), print production
