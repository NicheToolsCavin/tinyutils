Markdown is perfect for writing. But when you need to share a polished document, PDF is the universal format. Here's how to convert your .md files to PDF.

TL;DR

  • Open TinyUtils Document Converter
  • Upload your Markdown file (.md)
  • Select PDF as output
  • Click convert
  • Download the formatted PDF

Why convert Markdown to PDF?

  • Sharing — Everyone can open a PDF, not everyone knows Markdown
  • Printing — PDF preserves layout across printers
  • Archival — PDF is a long-term storage format
  • Signatures — PDF supports digital signatures
  • No editing — Recipients see it as you intended

What converts

  • Headings — # through ###### become styled headers
  • Lists — Bullets and numbered lists preserved
  • Bold/Italic — **bold** and *italic* formatting
  • Links — Clickable in the PDF
  • Code blocks — Syntax highlighting preserved
  • Tables — Markdown tables become PDF tables
  • Images — Referenced images embedded in PDF

How to convert

  1. Go to TinyUtils Document Converter
  2. Upload your .md file (or paste Markdown text)
  3. Select PDF from the output dropdown
  4. Click Convert
  5. Preview the result
  6. Download your PDF

Styling options

The converter applies sensible defaults:

  • Clean, readable font
  • Appropriate margins
  • Page breaks before major headings
  • Syntax highlighting for code

How images work

If your Markdown references images with relative paths like ![](images/diagram.png), make sure those images are included. The converter embeds them in the PDF.

For images hosted online, the converter fetches and embeds them.

Multiple files

Have several Markdown files? Upload them all and get a ZIP of PDFs. Each .md file becomes one PDF.

YAML frontmatter

If your Markdown has YAML frontmatter (title, author, date), it's processed: 

---
title: My Document
author: Jane Doe
date: 2025-01-15
---

# Introduction
...

The title and author appear in the PDF metadata.

FAQ

What Markdown flavor is supported?

GitHub Flavored Markdown (GFM). Tables, task lists, strikethrough, and fenced code blocks all work.

Can I customize fonts or colors?

The current version uses clean defaults. For advanced styling, convert to HTML first, add CSS, then convert that HTML to PDF.

Are links clickable in the PDF?

Yes. Both inline links and reference-style links become clickable.

What about math equations?

LaTeX math (like $x^2$) is rendered if present in the source.
