Sitemap changed? Don't guess what's different. Compare two versions and see exactly which URLs were added, removed, or modified.

TL;DR

  • Open TinyUtils Sitemap Delta
  • Enter your old and new sitemap URLs
  • Click Compare
  • See added/removed URLs
  • Export redirect rules

Why compare sitemaps?

  • Site migrations — Ensure all old URLs have redirects
  • Content audits — Track what was published or removed
  • SEO monitoring — Catch accidental deindexing
  • Staging validation — Verify changes before going live

How to compare sitemaps

  1. Go to TinyUtils Sitemap Delta
  2. Enter the "before" sitemap URL
  3. Enter the "after" sitemap URL
  4. Click Compare
  5. Review the results

What you get

  • Added URLs — Pages in new sitemap but not in old
  • Removed URLs — Pages in old sitemap but not in new
  • URL count — Total pages in each sitemap
  • Export options — CSV, JSON, or redirect rules

Generating redirect rules

For removed URLs, you often need redirects. Sitemap Delta can export:

  • nginx rewrite rules
  • Apache .htaccess rules
  • CSV for manual mapping

Common use cases

Site migration

Compare pre-migration and post-migration sitemaps. Every removed URL should have a redirect to avoid 404s.

CMS update

After a major CMS update, compare sitemaps to ensure nothing disappeared.

Content cleanup

If you removed old content, verify it's gone from the sitemap and set up proper redirects.

FAQ

Can I compare local sitemap files?

Currently, enter URLs. For local files, host them temporarily or paste the XML content.

Does it handle sitemap indexes?

Yes, sitemap indexes are expanded and all child sitemaps are processed.

What about sitemap.xml.gz?

Compressed sitemaps are supported. The tool handles gzip automatically.
