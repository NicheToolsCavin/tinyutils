Most online tools upload your files to their servers. That's a privacy risk. Here's how to find and use tools that process data locally.

TL;DR

  • Look for "browser-based" or "no upload" claims
  • Check network requests in browser dev tools
  • Prefer open-source tools where possible
  • TinyUtils processes images entirely in your browser

The privacy risk

When you upload a file to an online converter, that file goes to someone else's server. They might:

  • Store your files (intentionally or by accident)
  • Use them for training AI models
  • Be hacked, exposing your data
  • Share data with third parties

How local processing works

Privacy-first tools use JavaScript and WebAssembly to process files in your browser. Your files never leave your device. The server only serves the web page code — it never sees your data.

How to verify

  1. Open browser dev tools (F12)
  2. Go to the Network tab
  3. Process a file
  4. Check if any requests contain your file data

For truly local tools, you'll only see the initial page load — no file uploads.

TinyUtils privacy

Limitations of local processing

  • Device power — Your computer does the work
  • File size — Very large files may strain browser memory
  • Complex processing — Some operations need server power

When upload is OK

Not all uploads are bad. Consider:

  • Non-sensitive files — Public documents, stock photos
  • Trusted providers — Established companies with privacy policies
  • Temporary processing — Files deleted immediately after

FAQ

How do I know if a tool really doesn't upload?

Check the Network tab in dev tools. No outbound data = truly local.

Are desktop apps more private?

Not necessarily. Some desktop apps phone home. Local web apps can be just as private.

What about AI-powered tools?

AI typically requires server processing. If you need AI features, your data will be processed remotely.
